-> Documentação

    Os padrões de comentáripos são dados a uma boa intepretação:
        - O código deve ser escrito para humanos
        - Comentar é uma ferramenta adicional que um dev pode optar
        por usar ou não.
        - OS comentários fazer parte do código.
    
    -> boas práticas coodando:
        - Nomear melhor
        - Estrutura do cṕdigo
        - Comentários explicando determinada linha de código

    É preferivel um código com poucos comentários (evitar poluição visual)
    
    Boas práticas requer boa identação (pai/filho), espaço nos códigos (entre 
    linhas), nomes intuitivos de class, id, style, function, etc. Manter um 
    padrão de escrita.
        Ex: function exibirMensagem()
            class = "containerApp"
            var meuNome = "nome"

    Ao utilizar a nomenclatura adequada, podemos entender melhor o que o código faz.

    Aplicando diferentes ferramentas proporcionamos mais clareza, com diferentes graus
    de sucesso.

    Um exemplo básico de boa documentação:
        Um dev chega em um jnovo projeto e nesse projeto tem toda a documentação 
        de como o software funciona, sendo assim, ele perde menos tempo para
        aprender a utiliza-lo. Agora imagine, e se não houvesse uma documentação?

    Quando o código é mudado, o documento também precisa ser atualizado.
    Os comentários precisam de manutenção, ao refatorar o código, você
    também deve refatorar os comentários.

    Bons comentários são difíceis de escrever, bons comentários devem ser precisos
    e relevantes.

    -> Mantenedores:
        - O que o código faz
        - Como o código faz isso

        - Como usar o código
